اتعملت اعمل fetch data 
useState + useEffect: Hooks من React لإدارة الحالة وتنفيذ side effects.
axios: مكتبة لعمل طلبات HTTP (GET, POST …).
PokemonList: كومبوننت بتعرض قائمة البوكيمون.
Move: كومبوننت بتعرض أزرار التنقل (التالي / السابق).
//////////////////////////////////////////////////////////
pokemon: مصفوفة فيها أسماء البوكيمونات المعروضة.
loading: حالة التحميل (true في البداية لحد ما البيانات تيجي).
currentPage: اللينك الحالي اللي بيجيب منه البيانات (أول لينك هو API بتاع البوكيمون).
nextPage: لينك الصفحة التالية (الـ API بيرجعه في الـ response).
prevPage: لينك الصفحة السابقة.
////////////////////////////////////
: بيتنفذ كل مرة تتغير فيها قيمة currentPage.
AbortController: بيستخدم لإلغاء الطلب لو المستخدم غيّر الصفحة بسرعة أو خرج من الكومبوننت.
axios.get(currentPage): بيعمل طلب GET للـ API.
response.data.results.map((p) => p.name): بيجيب أسماء البوكيمونات بس ويحطها في الـ state pokemon.
setNextPage / setPrevPage: بيخزن اللينكات للصفحات التالية والسابقة.
return () => controller.abort(): Cleanup function → بتلغي أي طلب سابق عشان ميبقاش فيه memory leak أو clash في البيانات.
/////////////////////////////////////
goNextPage: يغير currentPage للصفحة التالية → ده يخلي useEffect يشتغل من تاني ويجيب البيانات الجديدة.
///////////////
الكود بيعمل Pagination للبوكيمونات باستخدام الـ API.

كل مرة المستخدم يضغط "التالي" أو "السابق"، بيتغير currentPage → useEffect بيشتغل → يعمل طلب جديد من الـ API → يحدث البيانات.

بيستخدم AbortController لإلغاء الطلب القديم لو حصل تغيير سريع.
